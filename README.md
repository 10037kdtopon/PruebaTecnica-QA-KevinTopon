# 🚀 **Ejercicio  QA - Kevin Topon**

## 📋 **Descripción**

Este repositorio contiene la solución de la **Prueba Técnica QA**, desarrollada aplicando buenas prácticas de **automatización de pruebas**, **validación de procesos** y **aseguramiento de la calidad del software**.

El objetivo fue demostrar conocimientos en **pruebas funcionales E2E** y **pruebas de servicios REST**, utilizando herramientas ampliamente utilizadas en la industria como **Serenity BDD** y **Karate Framework**.

Cada ejercicio incluye documentación, código fuente, conclusiones y los recursos necesarios para su ejecución y reproducción.

---

## 📂 **Contenido**

### ✅ **Ejercicio 1 - Automatización E2E Demoblaze**

**Ubicación:** `serenity-cucumber-starter/`

Automatización funcional **End-to-End** desarrollada con **Serenity BDD**, **Cucumber** y el patrón **Screenplay**.

#### **Flujo automatizado**

* Agregar dos productos al carrito.
* Visualizar el carrito.
* Completar el formulario de compra.
* Finalizar la compra.

#### **Documentación**

* `readme.txt`
* `conclusiones.txt`

---

### ✅ **Ejercicio 2 - API PetStore con Karate**

**Ubicación:** `karate-petstore/`

Automatización de pruebas **REST API** utilizando **Karate Framework** para validar el ciclo de vida completo de un usuario en la API PetStore.

#### **Casos implementados**

* Crear usuario.
* Buscar usuario creado.
* Actualizar nombre y correo electrónico.
* Buscar usuario actualizado.
* Eliminar usuario.

#### **Documentación**

* `readme.txt`
* `conclusiones.txt`
